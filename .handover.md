## How the System Works End-to-End

```
User hits a bug or false positive
        ↓
regula feedback false-positive --pattern X --file Y --line N
        ↓
Opens pre-filled GitHub Issue URL in browser
Label pre-filled: false-positive / false-negative / crash
        ↓
GitHub Actions: triage.yaml fires on issue open
Claude Haiku classifies → applies label automatically
        ↓
Every Monday 09:00 UTC: weekly-digest.yaml runs
Claude Sonnet reads all open issues → writes TRIAGE.md
        ↓
Maintainer opens TRIAGE.md → sees prioritised list → fixes
```

## New CLI Commands

```bash
# Report a false positive (pattern incorrectly flagged)
regula feedback false-positive --pattern HIGH_RISK_BIOMETRIC --file src/x.py --line 42

# Report a false negative (risk Regula missed)
regula feedback false-negative --pattern UNKNOWN --file src/y.py

# Report a crash / unexpected behaviour
regula feedback bug --description "crashed with AttributeError on scan"

# Check / change telemetry consent
regula telemetry status
regula telemetry enable
regula telemetry disable
```

All `feedback` subcommands accept `--no-browser` to print the URL instead of opening it (also auto-detected in CI).
