# EU AI Act Regulatory Delta Log

A versioned, primary-source-linked record of every change to the EU AI Act
and adjacent frameworks (Digital Omnibus on AI, delegated acts, Commission
guidance, national transpositions, harmonised standards, enforcement
actions).

**Why this exists.** Existing trackers (Future of Life Institute AI Act
Explorer, IAPP Regulatory Directory, AlgorithmWatch, Holistic AI) publish
blog-form updates or gated matrices. Nobody publishes a structured,
machine-readable, article-keyed, diff-able changelog with primary-source
links. This repository is that changelog. Regula's CLI reads it to warn
users when their installed ruleset is older than the last regulatory
change.

## Layout

```
content/regulations/delta-log/
├── README.md              ← you are here
├── schema.json            ← JSON Schema for every entry
├── index.json             ← auto-generated manifest (do not edit by hand)
└── entries/
    └── YYYY-MM-DD-slug.json
```

Each entry file conforms to `schema.json`. New entries are added by
opening a pull request with the entry file and a commit message of the
form `delta-log: <short description>`. CI validates every entry against
the schema and rebuilds `index.json`.

## Fields (summary)

See `schema.json` for the full specification.

- `id` — stable identifier, `YYYY-MM-DD-short-slug`
- `date` — publication or adoption date
- `type` — one of fourteen enumerated change types
- `source_url` — primary source (EUR-Lex CELEX preferred for EU acts)
- `celex_number` — EUR-Lex identifier when applicable
- `affected_articles` / `affected_recitals` — article-keyed references
- `member_state` — ISO 3166-1 alpha-2 code for national transpositions
- `summary` — one-paragraph plain-language description
- `mechanical_change` — what concretely changed (dates, thresholds)
- `impact_on_regula_patterns` — which of Regula's 330 patterns are
  affected and how (reviewed-no-change / updated / added / deprecated)
- `confidence` — verified-primary / verified-secondary / reported-
  unverified / speculative
- `verified_by`, `verified_on` — audit trail

## Editorial rules

1. **Primary source or nothing.** Aggregator blogs, consultancy summaries
   and news recaps are NOT primary sources. Cite EUR-Lex, the Commission
   press office, the Member State official gazette, or the court ruling
   directly.
2. **No speculation.** If a change is proposed but not adopted, set
   `type` to `amendment-proposal` and `confidence` to
   `reported-unverified`. Do not present proposals as if they were
   adopted.
3. **Article-keyed.** Every entry must list `affected_articles`. If the
   change affects the whole regulation, list the relevant chapters.
4. **Dated in UTC.** Use ISO 8601 dates.
5. **Every entry is commit-sized.** One change, one entry file, one
   commit. Do not batch multiple regulatory changes into a single
   commit.
6. **British English in prose.** Match EUR-Lex EN spelling.

## Reading the log programmatically

```bash
# Regula CLI — warn when the installed ruleset is older than the
# most recent delta-log entry that touches a pattern you use.
regula regwatch

# Emit as JSON
regula regwatch --format json

# RSS feed (auto-generated by scripts/build_delta_log.py)
curl https://getregula.com/content/regulations/delta-log/feed.xml
```

## Counterevidence and known gaps

- IAPP, Future of Life Institute and OneTrust could all publish a
  structured tracker. The moat here is being first, being open-source,
  and being maintained by developers who can wire it directly into the
  scanning tool.
- This log is not a substitute for legal advice. It is a citation
  infrastructure for developers and compliance teams who need to know
  *what* changed and *where*, not *whether* the change applies to their
  specific system.
- Confidence labels are not guarantees. Every entry traces to a
  primary-source URL that the reader can verify independently.

## How to contribute

1. Fork the repository.
2. Copy an existing entry from `entries/` as a template.
3. Fill in the fields, citing the primary source URL.
4. Validate locally: `python3 scripts/build_delta_log.py --validate`
5. Open a PR. CI will re-validate and rebuild `index.json`.
