# Regula: AI Governance for Code — Interactive Course

Learn AI governance by building with Regula, an open-source tool that scans code for EU AI Act risk patterns.

## What This Course Covers

You'll learn to scan AI codebases for regulatory compliance issues, understand what the EU AI Act requires from AI systems, and use code analysis to generate compliance evidence.

## Prerequisites

- Python 3.10 or later
- git
- A code editor
- Basic familiarity with Python (you don't need to be an expert)

## Course Structure

| Module | Topic | Time |
|--------|-------|------|
| [1. Setup](01-setup.md) | Install Regula and run your first scan | 15 min |
| [2. Risk Classification](02-risk-classification.md) | How the EU AI Act classifies AI systems | 30 min |
| [3. Scanning Real Code](03-scanning-real-code.md) | Scan open-source AI projects and interpret results | 30 min |
| [4. Compliance Gaps](04-compliance-gaps.md) | Assess what's missing from an AI project's compliance infrastructure | 30 min |
| [5. Dependency Security](05-dependency-security.md) | Check AI supply chain pinning and known compromised packages | 20 min |
| [6. AI Security Patterns](06-ai-security-patterns.md) | Detect unsafe deserialization, prompt injection, eval on AI output | 20 min |
| [7. CI/CD Integration](07-cicd-integration.md) | Add Regula to your GitHub Actions pipeline | 20 min |
| [8. Documentation Generation](08-documentation.md) | Generate Annex IV and QMS scaffolds from code analysis | 20 min |
| [9. Framework Mapping](09-framework-mapping.md) | Map findings to NIST, ISO, OWASP, and MITRE frameworks | 15 min |
| [10. Building Your Own Patterns](10-custom-patterns.md) | Extend Regula with domain-specific detection rules | 30 min |

**Total estimated time: ~4 hours**

## How to Use This Course

Each module is self-contained. You can work through them in order or jump to what interests you. Every module includes:
- Concepts (what and why)
- Hands-on exercises (run real commands)
- Verification (check your understanding)

Start with [Module 1: Setup](01-setup.md).
