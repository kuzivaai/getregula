Shipping AI to production

Developer deploying AI features to the EU market?

Regula maps your code to the exact EU AI Act articles, shows which risk tier you're in, estimates the compliance effort per article, and generates the documentation auditors need.

2026–27
High-risk obligations start August 2026 — or December 2027 if the Digital Omnibus passes (Parliament voted 569–45 in favour, March 2026). Either way, the requirements don't change. You need to know where you stand.
What you get

Run one command. Get answers you can act on.

1. Your risk level

Prohibited, high-risk, limited, or minimal — with the reason why

Points to the exact line of code that triggered the classification, cites the legal basis, and tells you when it might be a false positive. No guessing.

regula check . --explain
2. What you need to do

A prioritised list of compliance tasks with effort estimates

If you're high-risk, the EU AI Act requires specific actions (risk management, logging, human oversight, documentation). Regula tells you exactly which ones apply and how long each takes.

regula plan --project .
3. Documentation that writes itself

Technical documentation pulled from your actual code

Reads your dependencies, function signatures, and logging coverage. Generates the documentation skeleton that the law requires. You fill in the business context.

regula docs --completion
4. Proof for auditors

One command, one folder — everything an auditor needs

Bundles scan results, gap assessment, documentation draft, dependency report, and audit trail into a single reviewable package with integrity verification.

regula evidence-pack --project .
What the law covers — and what Regula checks

Not a questionnaire. Regula reads your code and maps it to the regulation.

Banned uses

Is your app prohibited?

Social scoring, subliminal manipulation, real-time facial recognition, emotion inference in workplaces — all banned outright.

High-risk uses

Does your app make important decisions?

Credit scoring, hiring, medical diagnosis, education assessment, law enforcement, border control — all classified as high-risk with strict requirements.

Compliance gaps

What’s missing from your codebase?

Risk management, data governance, logging, transparency, human oversight, accuracy — scored per article with effort estimates.

Documentation

Can you prove compliance?

Auto-generates technical documentation from your actual code — functions, dependencies, and logging coverage already filled in.

Security

Is your AI code secure?

Checks for prompt injection, unsafe model loading, unvalidated AI output, hardcoded keys, and other AI-specific vulnerabilities.

Transparency

Do your users know it’s AI?

Chatbots must disclose they’re not human. Deepfakes must be labelled. Regula checks if your code handles this.
