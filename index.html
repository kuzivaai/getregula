The gap

Governance tooling
stops at the policy layer.

Every major AI compliance platform — Credo AI, IBM OpenScale, Holistic AI — operates at the metadata and workflow level. They manage assessments, track documentation, and run audits. None of them read your code.

By the time your AI system reaches a compliance review, prohibited patterns may already be baked into production. Regula moves the check to the only place it can reliably prevent problems: the moment the code is written.

It runs as a real-time hook inside your coding assistant. It scans entire codebases from the CLI. It logs everything to a hash-chained audit trail and generates the Annex IV documentation your DPO needs.

How it works

Install once. Runs everywhere.

One guided wizard handles platform detection, hook registration, and your first scan. No configuration required.

01 / Install
⚙️

One command setup

Guided wizard detects your platform, installs hooks, and runs your first scan. Works offline. Zero required dependencies.

pip install -e .
regula init
# detects platform, installs hooks
02 / Scan
🔍

Hooks fire on every write

Pre-write hooks intercept code before it's saved. Or scan an entire project on demand. SARIF output for CI/CD pipelines.

regula check .
regula check . --ci
# exits 1 on any finding
03 / Evidence
📄

Audit trail & docs

Every finding logs to a hash-chained, tamper-evident audit trail. Generate Annex IV documentation and Articles 9–15 gap reports.

regula gap --project .
regula docs --all
# Annex IV + QMS scaffolds

Get started

Install in under two minutes.

Three ways to install. The guided wizard handles the rest — platform detection, hook registration, first scan.

Recommended — pip
bash
pip install -e .
regula init   # guided setup
From source
bash
git clone https://github.com/kuzivaai/getregula.git
cd getregula
python3 scripts/cli.py init
Manual hook install
bash
regula install claude-code   # Claude Code
regula install copilot-cli   # GitHub Copilot CLI
regula install windsurf      # Windsurf Cascade

Requirements

  • Python 3.10 or later
  • No external dependencies required
  • Linux, macOS, and Windows
  • Works completely offline

Optional enhancements

  • pyyaml — full YAML policy config
  • tree-sitter — deep JS/TS AST analysis
  • pytest — run the test suite (362 tests)

ℹ️

What Regula is: A pattern-based risk indicator. It scans for code patterns associated with prohibited and high-risk AI practices under Regulation (EU) 2024/1689.

What it is not: A legal determination of risk classification. Article 6 classification requires human judgment about intended use and deployment context. False positives and negatives are inherent to pattern matching.

What to do next: Share findings with your DPO or legal counsel. The audit trail is designed to support that conversation.
